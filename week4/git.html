GIT

Powrót do strony głównej

Komendy

  • git config --global user.name "Jan Nowak" - ustawienie nazwy twórcy repo
  • git config --global user.email jannowak@example.com - ustawienie adresu e-mail twórcy repo

Repozytorium = projekt

czyli wszystkie pliki

  • git init - tworzy repo w wybranym folderze
  • git status - sprawdza stan projektu
  • git add . - dodaje wszystkie pliki do projektu
  • git commit -m "opis zmian" - dodaje commit z opisem
  • gitk - wyświetla okno, w którym możemy podejrzeć commity
  • git push - wypchnięcie kodu na serwer

Podsumowanie

  • GIT pozwala na sprawne zarządzanie zmianami w projekcie
  • Działa lokalnie
  • Do działania zdalnego potrzebuje serwera (np. GitHub)
  • Znajomość podstaw GIT-a jest wymagana niezależnie od technologii
  • Konto na GitHubie jest Twoją programistyczną wizytówką, zadbaj aby była ładna i prawdziwa :)
  • Commitujemy jak dokładamy jakąś widoczną zmianę do projektu (lepiej częściej, niż za mało
  • Podstawowy flow: git add . / git commit -m "opis zmiany" / git push
